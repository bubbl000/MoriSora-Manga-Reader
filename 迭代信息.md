```rust
            .unwrap_or_default();

```

##### 4.1 natural_cmp 正则表达式缓存（sort_utils.rs）

**问题**：每次字符串比较都重新编译正则表达式
**优化**：使用 `std::sync::LazyLock` 缓存静态正则表达式

```rust
// 优化前
pub fn natural_cmp(a: &str, b: &str) -> Ordering {
    let re = regex::Regex::new(r"(\d+)").unwrap(); // 每次编译
}

// 优化后
static NUMBER_RE: LazyLock<Regex> = LazyLock::new(|| Regex::new(r"(\d+)").unwrap());

pub fn natural_cmp(a: &str, b: &str) -> Ordering {
    let re = &*NUMBER_RE; // 复用缓存
}
```

**预期收益**：排序时间减少 60-80%

##### 4.2 count_manga_in_folder SQL LIKE 优化（database.rs + main.rs）

**问题**：全量获取漫画列表到内存，然后在 Rust 端过滤统计
**优化**：直接在 SQL 中使用 LIKE 查询统计

```rust
// 新增函数
pub fn count_comics_in_folder(state: &AppState, folder_path: &str) -> Result<usize, String> {
    let pattern = format!("{}%", folder_path.trim_end_matches('\\'));
    let count: i64 = conn_guard.query_row(
        "SELECT COUNT(*) FROM comic_metadata WHERE path LIKE ?1",
        params![pattern],
        |row| row.get(0),
    )?;
    Ok(count as usize)
}
```

**预期收益**：查询性能提升 100-1000 倍，数据传输减少 99%+

##### 4.3 batch_upsert 使用 RETURNING 子句（database.rs）

**问题**：每条记录 INSERT 后需要额外 SELECT 查询 ID
**优化**：使用 SQLite 3.35+ 的 RETURNING 子句

```sql
-- 优化前

-- 优化后
```

**预期收益**：SQL 执行次数减半，批量插入速度提升 40-50%

```typescript
class LRUCache<K, V> {
  private cache: Map<K, V>
  private readonly maxSize: number
  private readonly onEvict?: (value: V) => void

  set(key: K, value: V): void {
    if (this.cache.has(key)) this.cache.delete(key)
    this.cache.set(key, value)
    // 超出容量时淘汰最旧条目
    while (this.cache.size > this.maxSize) {
      const oldestKey = this.cache.keys().next().value
      if (oldestKey !== undefined) {
        const oldestValue = this.cache.get(oldestKey)
        if (oldestValue) this.onEvict?.(oldestValue)
        this.cache.delete(oldestKey)
      }
    }
  }
}
```

- 实现300ms节流（WHEEL_THROTTLE_MS = 300）

- 清理时取消pending定时器

- 使用`{ passive: false }`确保e.preventDefault()生效
- 避免浏览器忽略preventDefault导致滚动穿透

| 滚动穿透 | passive: true时无法阻止滚动 | 设置passive: false |

### 📐 架构决策

| 决策项 | 选择 | 原因 |
|--------|------|------|
| 节流时间 | 300ms | 平衡响应速度和防误触 |
| 滚轮方向判断 | deltaY > 0为下一页 | 符合用户直觉，向下翻下一页 |
| passive选项 | false | 需要调用preventDefault阻止滚动 |

### 📝 迭代内容

#### 1. perf.rs 模块扩展

- `CacheStats` 结构体记录 zip_hits/zip_misses/cbr_hits/cbr_misses
- `zip_hit_rate()` / `cbr_hit_rate()` 计算命中率百分比

- `get_or_load_zip`: 命中时 zip_hits++，未命中时 zip_misses++
- `get_or_extract_cbr`: 命中时 cbr_hits++，未命中时 cbr_misses++
- 两个函数均添加 `perf::start_timer()` / `perf::stop_timer()` 计时

**预期收益**：可量化验证 LRU 缓存优化的有效性，命中率应 >70%

#### 3. read_image_bytes 图片大小记录

**修改 main.rs 中的 read_image_bytes 函数**：
- 读取成功后调用 `perf::record_image_size(data.len() as u64)`
- 实时输出 `[IMAGE] {path} size: {xx.xx}KB` 日志
- 配合 `perf::start_timer("read_image_bytes")` 记录耗时

**预期收益**：实时输出每张图片的文件大小，便于发现异常大文件

## 迭代 11：自动翻页功能

**日期**: 2026-04-20  
**状态**: ✅ 已完成

### 🎯 本次迭代目标

为阅读器添加自动翻页功能，支持自定义翻页间隔，适用于单页和双页模式。

### 📝 迭代内容

#### 1. 自动翻页状态管理
- 新增autoPage状态（布尔值，控制自动翻页开关）
- 新增autoPageInterval状态（数值，默认3000ms，范围500ms-10000ms）
- 新增autoPageRef（ref存储setInterval ID，避免闭包问题）

#### 2. 自动翻页定时器
- 使用setInterval实现定时翻页
- 间隔内自动调用setCurrentPage递增页码
- 单页模式每次+1，双页模式每次+2
- 到达最后一页时自动停止
- 用户手动翻页时自动停止

#### 3. 页码变化联动
- autoPage模式下，currentPage变化时自动加载对应页面
- 复用已有的loadDoublePage/switchToPage逻辑
- 新增useEffect监听currentPage变化触发页面加载

#### 4. 工具栏UI
- 新增自动翻页按钮：「自动开」/「自动关」
- 激活状态：亮绿色accent背景
- 开启后显示间隔调节控件：
  - 「-」按钮：减少500ms（最小500ms）
  - 「+」按钮：增加500ms（最大10000ms）
  - 中间显示当前间隔（如「3.0s」）
- 滚动模式下隐藏自动翻页功能

#### 5. 交互优化
- 手动点击上一页/下一页时自动关闭自动翻页
- 到达最后一页自动停止
- 切换阅读模式时保留自动翻页状态
- 组件卸载时正确清理定时器

### ⚠️ 遇到的问题与解决（重点记录）

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| setInterval内无法获取最新state | 闭包捕获旧值 | 使用setCurrentPage函数式更新 |
| 手动翻页后自动翻页继续 | 状态不同步 | handlePrev/Next内调用setAutoPage(false) |
| 页面加载不触发 | currentPage变化未触发useEffect | 新增useEffect专门监听currentPage变化 |

### 📐 架构决策

| 决策项 | 选择 | 原因 |
|--------|------|------|
| 定时器存储 | useRef | 避免闭包问题，setInterval ID不需要触发重渲染 |
| 间隔调节 | ±500ms步进 | 精细调节，覆盖500ms-10s范围 |
| 页码递增 | setCurrentPage函数式更新 | 不需要依赖外部state，避免闭包陷阱 |
