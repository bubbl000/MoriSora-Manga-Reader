## 迭代 2：多窗口架构实现与问题修复

**日期**: 2026-04-19  
**状态**: ✅ 已完成

### 🎯 本次迭代目标

实现标准桌面阅读器架构：书库为主窗口，双击漫画打开独立的阅读器窗口，支持多窗口同时运行。

### 📝 迭代内容

#### 1. 多窗口架构实现
- 采用前端 `WebviewWindow` 创建新窗口（参考 comic-shelf-main）
- 实现双击漫画卡片打开独立阅读器窗口
- 支持同时打开多个阅读器窗口
- 使用 hash 路由传递漫画信息（`/#reader#{id}#{title}`）

#### 2. 权限配置
- 配置 Tauri v2 capabilities 权限系统
- 添加 `core:window:allow-close` 关闭窗口权限
- 添加 `core:webview:allow-create-webview-window` 创建窗口权限

#### 3. 组件拆分
- 创建独立组件：`LibraryView`（书库）和 `ReaderView`（阅读器）
- App.tsx 根据 URL hash 自动切换书库/阅读器模式
- 阅读器窗口包含关闭按钮、页码显示、翻页控制

#### 4. 问题修复与优化

### ⚠️ 遇到的问题与解决（重点记录）

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 阅读器窗口空白 | Rust 后端创建窗口时 URL 格式问题，Vite dev server 无法处理 query 参数 | 改为前端 `new WebviewWindow()` 创建窗口 |
| 关闭按钮无效 | 渲染进程挂起导致 Tauri IPC 通道未初始化 | 修复窗口创建方式后自动解决 |
| 关闭窗口后主窗口正常但阅读器未响应 | 阅读器 IPC 从未建立，主窗口关闭后进程断开 | 同上 |
| 双击漫画显示书库而非阅读器 | URL hash 格式不正确，App.tsx 检测逻辑不匹配 | 统一 URL 格式为 `/#reader#{id}#{title}` |

### 📐 架构决策变更

| 决策项 | 迭代1方案 | 迭代2方案 | 原因 |
|--------|-----------|-----------|------|
| 窗口创建方式 | Rust 后端 `WebviewWindowBuilder` | 前端 `new WebviewWindow()` | 参考 comic-shelf-main，前端创建确保 IPC 正确初始化 |
| 参数传递方式 | query 参数 `/?window_type=reader&...` | hash 路由 `/#reader#{id}#{title}` | SPA 标准做法，Vite 不拦截 hash |
| 窗口关闭方式 | Rust 命令 `close_current_window` | 前端 `getCurrentWindow().close()` | 权限配置正确后直接调用 API |
