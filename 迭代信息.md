## 迭代 2：多窗口架构实现与问题修复

**日期**: 2026-04-19  
**状态**: ✅ 已完成

### 🎯 本次迭代目标

实现标准桌面阅读器架构：书库为主窗口，双击漫画打开独立的阅读器窗口，支持多窗口同时运行。

### 📝 迭代内容

#### 1. 多窗口架构实现
- 采用前端 `WebviewWindow` 创建新窗口（参考 comic-shelf-main）
- 实现双击漫画卡片打开独立阅读器窗口
- 支持同时打开多个阅读器窗口
- 使用 hash 路由传递漫画信息（`/#reader#{id}#{title}`）

#### 2. 权限配置
- 配置 Tauri v2 capabilities 权限系统
- 添加 `core:window:allow-close` 关闭窗口权限
- 添加 `core:webview:allow-create-webview-window` 创建窗口权限

#### 3. 组件拆分
- 创建独立组件：`LibraryView`（书库）和 `ReaderView`（阅读器）
- App.tsx 根据 URL hash 自动切换书库/阅读器模式
- 阅读器窗口包含关闭按钮、页码显示、翻页控制

#### 4. 问题修复与优化

### ⚠️ 遇到的问题与解决（重点记录）

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 阅读器窗口空白 | Rust 后端创建窗口时 URL 格式问题，Vite dev server 无法处理 query 参数 | 改为前端 `new WebviewWindow()` 创建窗口 |
| 关闭按钮无效 | 渲染进程挂起导致 Tauri IPC 通道未初始化 | 修复窗口创建方式后自动解决 |
| 关闭窗口后主窗口正常但阅读器未响应 | 阅读器 IPC 从未建立，主窗口关闭后进程断开 | 同上 |
| 双击漫画显示书库而非阅读器 | URL hash 格式不正确，App.tsx 检测逻辑不匹配 | 统一 URL 格式为 `/#reader#{id}#{title}` |

### 📐 架构决策变更

| 决策项 | 迭代1方案 | 迭代2方案 | 原因 |
|--------|-----------|-----------|------|
| 窗口创建方式 | Rust 后端 `WebviewWindowBuilder` | 前端 `new WebviewWindow()` | 参考 comic-shelf-main，前端创建确保 IPC 正确初始化 |
| 参数传递方式 | query 参数 `/?window_type=reader&...` | hash 路由 `/#reader#{id}#{title}` | SPA 标准做法，Vite 不拦截 hash |
| 窗口关闭方式 | Rust 命令 `close_current_window` | 前端 `getCurrentWindow().close()` | 权限配置正确后直接调用 API |

## 迭代 3：文件解析系统与多格式支持

**日期**: 2026-04-19  
**状态**: ✅ 已完成

### 🎯 本次迭代目标

实现完整的文件解析系统，支持 CBZ/ZIP/CBR/RAR/PDF 格式漫画文件，让阅读器能够打开各种格式的漫画。

### 📝 迭代内容

#### 1. 目录扫描功能
- 实现 `library_scanner.rs` 模块，扫描指定目录中的漫画文件
- 支持识别图片文件夹、CBZ/ZIP/CBR/RAR 压缩包、PDF 文件
- 实现自然排序算法，正确排序漫画文件名

#### 2. 压缩包解析模块
- 创建 `archive_parser.rs` 模块（参考 comic-shelf-main）
- 实现 CBZ/ZIP 格式解析（使用 `zip` crate）
- 实现 CBR/RAR 格式解析（使用 `unrar` crate）
- 实现 PDF 页数统计（使用 `lopdf` crate）

#### 3. 多格式支持
- 添加 Rust 依赖：`unrar = "0.5"`, `lopdf = "0.35"`, `regex = "1"`
- 添加前端依赖：`pdfjs-dist = "^5.6.205"` 用于前端PDF渲染
- 实现 `read_image_bytes` 命令用于读取任意文件字节
- 实现 `get_archive_images` 命令获取漫画页面列表
- 实现 `get_archive_image_bytes` 命令读取压缩包内图片

#### 4. PDF渲染实现
- 在前端使用 pdfjs-dist 渲染PDF页面
- 使用 Canvas 将PDF页面渲染为图片
- 通过 FileReader.readAsDataURL() 生成图片URL
- 实现完整的PDF资源清理逻辑（destroy/cleanup）

#### 5. 设置功能
- 实现 `settings.rs` 模块
- 支持添加/移除漫画库路径
- 实现设置持久化存储

#### 6. 问题修复

### ⚠️ 遇到的问题与解决（重点记录）

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| PDF显示"暂无图片" | Rust后端扩展名匹配区分大小写，Windows上".PDF"无法匹配"pdf" | 在 `extension_of` 函数中将扩展名统一转换为小写 |
| PDF worker加载失败 | 静态导入worker文件导致路径问题 | 改为动态导入 `import('pdfjs-dist/build/pdf.worker.min.mjs?url')` |
| PDF页面无法渲染 | 使用 `URL.createObjectURL()` 生成Blob URL不稳定 | 改用 `FileReader.readAsDataURL()` 生成DataURL |
| PDF资源泄漏 | 未正确清理PDF文档和加载任务 | 添加 `pdf.destroy()` 和 `loadingTask.destroy()` 调用 |

### 📐 架构决策

| 决策项 | 选择 | 原因 |
|--------|------|------|
| PDF渲染位置 | 前端（pdfjs-dist） | Rust端lopdf只有解析功能，无渲染能力 |
| 图片URL生成 | FileReader.readAsDataURL() | 比BlobURL更稳定，与comic-shelf-main一致 |
| 自然排序 | 正则表达式匹配数字 | 正确处理"page1", "page2", "page10"等排序 |

- ✅ CBZ/ZIP格式支持
- ✅ CBR/RAR格式支持
- ✅ PDF格式支持（前端渲染）
- ✅ 自然排序算法

## 迭代 4：数据持久化系统

**日期**: 2026-04-19  
**状态**: ✅ 已完成

### 🎯 本次迭代目标

实现SQLite数据库持久化，存储漫画元数据、阅读进度、收藏和标签信息。

### 📝 迭代内容

#### 1. 数据库模块 (database.rs)
- 使用 rusqlite 进行SQLite操作
- 使用 lazy_static 实现全局数据库连接池
- 数据库文件存储在系统应用数据目录
- 启用WAL日志模式提升性能

#### 2. 数据库表设计
- `comic_metadata` - 漫画元数据（路径、标题、格式、页数等）
- `reading_progress` - 阅读进度（当前页、总页数）
- `favorites` - 收藏记录
- `tags` - 标签定义
- `comic_tags` - 漫画与标签关联表

#### 3. Tauri命令接口
- `init_db` - 初始化数据库
- `save_comic_metadata` - 保存漫画元数据
- `get_all_comics_metadata` - 获取所有漫画
- `update_comic_last_opened` - 更新最后打开时间
- `save_reading_progress` - 保存阅读进度
- `get_reading_progress` - 获取阅读进度
- `add_to_favorites` / `remove_from_favorites` - 收藏管理
- `is_favorite` / `get_favorite_comics` - 收藏查询
- `add_tag_to_comic` / `remove_tag_from_comic` - 标签管理
- `get_comic_tags` / `get_all_tags` - 标签查询

#### 4. 前端服务层 (databaseService.ts)
- 封装所有数据库操作的TypeScript接口
- 提供强类型的数据结构定义

#### 5. 状态管理集成 (mangaStore.ts)
- 扫描时自动保存漫画元数据到数据库
- 新增 `updateReadingProgress` 方法保存阅读进度
- 新增 `toggleFavorite` 方法管理收藏
- MangaItem接口扩展：isFavorite、currentPage、totalPages

### 📐 架构决策

| 决策项 | 选择 | 原因 |
|--------|------|------|
| 数据库位置 | 系统应用数据目录 | Windows: %APPDATA%, macOS: ~/Library/Application Support |
| 连接管理 | lazy_static全局单例 | 简化连接管理，避免重复打开 |
| 日志模式 | WAL | 提升并发读写性能 |
| 外键约束 | 启用 | 保证数据一致性，级联删除 |
